Bubble Sort

The bubble sort algorithm is in my opinion the easiest sorting algorithm to understand and conceptualize. Like any sorting algorithm, the purpose is to sort the values in a desired sequence and in this case we will be sorting the values from smallest to largest or in other words, natural ordering.
Bubble sort achieves this task by looking at the first two values in an array and comparing which value is larger. Since we are sorting from smallest to largest, we want the smaller values to be on the left hand side and the larger values to be on the right hand side. If the left value is smaller than the right value then we know that the pair is in the correct order and can move onto the next pair. If the left value is greater than the right value then we have to swap the placement of the values before moving onto the next pair. The next pair will be the indexes of the previous pairs plus 1, thus one of the values in the old pair will be compared in the new pair. This pair comparison is repeated until reaching the end of the array where the largest value in the array will “bubble” to the end of the array. Now this entire procedure is repeated until all the values in the array have been sorted.
In the following example, we start off by comparing the first two elements in the array. The reference pointer “Left” is targeting index 0 of the array while the pointer “Right” is at index 1. We compare whether the left value is less than the right value and if not then we swap the positions of the values with one another.
In this case, since 3 is not less than 1 we swap the values.
After comparing the first pair, we take a look at the next pair which are at index 1 and index 2. Since 3 is smaller than 5 we can move onto the next pair since these values are in the correct positions.
Now we are comparing index 2 with index 3. Since 5 is greater than 4 we know that it should be on the right side and 4 should be on the left side so we must swap the two values.
We now compare the last pair and have to swap the values since 5 is greater than 2.
We have now reached the end of our array and the result is that 5 has landed in its correct spot in the array since it is the largest value in the array and thus must be at the end or the array. We can now repeat the same steps until each of the next largest values continue to bubble to the right side of the array. This will eventually result in our array being completely sorted.
Note that for the last comparison, when we determine the correct value for index 1 it automatically determines the placement for index 0. Knowing this, we can save ourselves an additional iteration.

Code Implementation

The bubble sort algorithm is pretty easy to implement since we are just iterating through the array and swaping the values of pairs if they are not in the correct order. The main trick to this algorithm is knowing how many times we need to iterate through the array.
When we think about our loop structure, we know that we need one for loop that iterates through the array and bubbles the largest value to the end of the array. This will be the inner loop. Now, how many times will we need to repeat the bubbling of the largest value that has not been placed in its correct spot? The answer is simple, we will have to repeat this N times where N is the number of elements in the array. The reason for this is that each time we run through the inner loop we are only placing one of the values in the correct position. Since we want all of the values to be in the correct position we will need an outer loop that runs N number of times. Now we can optimize this a little since we know that running the Nth time and Nth - 1 time will be the same operation so really we only need to run the outer loop N - 1 times.

Now comes the interesting part of the algorithm. We need to compare each of the pairs and check whether the left index is smaller or greater than the right index, then swap the values if needed. We know that we want our left pointer to start at index 0 and our right pointer to start at index 1. We can utilize j to accomplish this by subtracting 1 from it to get the left pointer and having j be the right pointer. As j increases so will the left and right pointers.
Once we have established how to get our left and right pairs, we now need to check if the left value is smaller than the right value. We can do this with an if statement. Swapping the pairs is now a simple operation.

Time Complexity

It is always beneficial to consider the time complexity of a function because a function that can perform its task faster than another function is always better (assuming space complexity is the same). Bubble sort is a very straight forward algorithm to understand and implement, however if we analyze the time that it takes to run we will realize that it is pretty inefficient due to the fact that we are iterating through our array multiple times. We iterate through our array to bubble the largest value to the right hand side and repeat this operation for the number of elements in the array. This nested looping results in an O(N^2) time complexity which is quadratic. As we explore other sorting algorithms we will find that O(N^2) time is pretty slow since there are other sorting algorithms that can run in O(N*log N) time complexity such as quick sort.
